# My Adventures in AI Development

I’m a former (reformed?) analyst and data operations leader turned product builder. 

Recently I’ve been spending my free time trying to figure out how to actually build with AI, rather than just reading about it or passing off requirements to someone else. 

I've always loved turning ideas into something real and tangible, and now AI has progressed to the point where that speed and capability is available to those willing to learn. So, wanting to truly understand how  technology behaves—where it shines, where it falls apart, or how to design products around it- I decided to get my untrained hands dirty and build/break a few things myself. 

For me, these projects are less about writing code and more about learning; they are experiments to test different development workflows and learn-by-doing with AI. 

---

## What I'm Tinkering With

* **AI Orchestration:** Trying different stacks and levels of AI-driven workflows
* **New Skills:** Getting deep into new languages and tools like Python, React/Vite, building native vs. PWAs, and running my own CI/CD pipelines.
* **AI Concepts:** Model Context Protocol (MCP), custom agent tools/skills, multi-agent orchestration, and local execution capabilities.
* **Workflows:** Testing different models of architecture, scaffolding, vibe-coding, etc to learn different workflows. 

---

## The Build Log

Instead of just pinning repositories, I’m keeping a running log of my projects. Each entry includes a quick write-up of what I built, the specific tech stack, the direction I took with the project, and key learnings.
And no, I'm not posting the code...yet.  

### [Project TAP (Time and Place)](./ProjectTAP.md)
* **The Experiment:** A mobile-first Progressive Web App (PWA) for location-aware task management. I wanted to solve a personal problem with this project, and took direct feedback from a small group of Beta users to test my ability to iterate and prove feature-value in an AI-driven product.
* **Built In:** React + Vite using Antigravity IDE
* **The Collaboration Manifest:** *High human guidance / AI as an architect.* I kept tight control over the product UX and user flow, leaning into an agentic IDE that used AI agents to handle the structural scaffolding and software development. 
* [Read the project notes and what I learned →](./ProjectTAP.md)

### [Lord of the Keys](./Lord-of-the-Keys.md)
* **The Experiment:** A simple platformer expansion inspired by a friend's use of Codex to build a spelling game for their student. Designed for learning typing skills in a fun Lord of the Rings inspired game.
* **Built In:** HTML using Codex and Antigravity
* **The Collaboration Manifest:** *Pair programming.* I used local execution and treated the AI agent as a specialized technical soundboard with a time limit of 8 hours, mapping out requirements and rapidly prototyping mechanics together in real-time.
* [Read the project notes and what I learned →](./Lord-of-the-Keys.md)

### [Project Asgard]
* **The Experiment:** Inspired by problems-worth-solving in my professional life (construction), I built a series of vibe-coded point solutions to test my one-shot prompting capabilities and gaps.
* **Built In:** Angular + Vite, with Tailwind and other frameworks (different apps) using Google AI Studio
* **The Collaboration Manifest:** *Low human guidance / High AI autonomy.* I described a problem worth solving, the jobs to be done by the user, and some key features...then let the vibes run.


### [Custom Agentic Skills & Tooling]
* **The Experiment:** A deep dive into extending what AI agents can actually do by building custom tools and integrating global skills using the Model Context Protocol (MCP).
* **Built In:** Markdown using Gemini, VS Code,  Antigravity IDE
* **The Collaboration Manifest:** *Low human guidance / High AI autonomy.* I set the environmental boundaries and constraints, then let multi-agent orchestration figure out how to build and configure its own execution tools.
